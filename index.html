Agent Ready React SDK

一套面向 React 应用的声明式元数据与运行时契约层， 让 AI Agent 能够发现你的 UI、理解语义、 执行用户意图等价操作，并订阅结构化状态—— 无需依赖 DOM 抓取或脆弱的 RPA。

GitHub 仓库 API 文档

这个项目是做什么的？

现代 AI Agent（IDE Agent、自主 Agent、MCP Client、Eval Bot）需要与 Web 应用交互， 但传统方式——解析 DOM 文本、模拟点击——既不稳定，也难以审计和治理。

Agent Ready React SDK 在 React 应用与外部 Agent 之间插入一层 显式契约：你在组件里声明 Surface（可操作区域）、Action（可执行操作）、 Observation（可读状态），SDK 负责注册、编目、校验、派发与策略控制。 

External Agent  →  Bridge / MCP  →  Runtime  →  React Hooks  →  你的业务 UI

SDK 不包含业务逻辑与 LLM 运行时；它只提供 Agent 可见的「机器接口」。

核心能力

可发现 Discoverable

Agent 通过 Catalog 枚举页面/区域内的 Surface、Action 与 Observation，带分页与过滤。

可理解 Understandable

一切对外契约以 Zod / JSON Schema 为单一事实来源，输入输出可机器校验。

可执行 Actionable

useAgentAction(handle, action, handler) 将 Agent 调用路由到你写的 React 处理器。

可观测 Observable

useAgentObservation 暴露结构化快照，Agent 订阅状态变化而非 DOM 文本。

可治理 Governable

内置 Policy、OIDC、租户白名单、速率限制与审计日志，执行权始终在宿主应用。

RSC 友好

Server Component 通过 @agent-ready/react/rsc 声明 manifest，客户端 hydration 注入 Catalog。

为什么选择它？

  • Schema-first — 契约即文档，Agent 与人都读同一份类型定义。
  • Opt-in by default — 未注册的 UI 对 Agent 不可见，避免意外泄露。
  • 渐进式接入 — 按路由/Feature 切片接入，无需重写整个应用。
  • Host owns authority — 校验、策略、审计在应用内完成，不依赖外部黑盒。
  • Deterministic — 稳定 AgentHandle 寻址，版本化契约，拒绝启发式 DOM 推断。
  • 生态就绪 — MCP 传输、OpenTelemetry、CLI codegen、Playground 与 Kitchen Sink 示例齐全。

最小 React 集成示例：

import { AgentReadyProvider, useAgentSurface, useAgentAction } from "@agent-ready/react";
import { defineAction } from "@agent-ready/schema";
import { z } from "zod";

const handle = "app://forms/contact/main" as const;

function ContactForm() {
  useAgentSurface({ handle, title: "Contact Form", capabilities: ["act"] });

  useAgentAction(
    handle,
    defineAction({
      name: "submitForm",
      description: "Submit the contact form",
      input: z.object({ email: z.string().email() })
    }),
    async (input) => ({ submitted: true, email: input.email })
  );

  return <form>{/* your UI */}</form>;
}

export function App() {
  return (
    <AgentReadyProvider session={{ sessionId: "web-1", roles: ["agent"] }}>
      <ContactForm />
    </AgentReadyProvider>
  );
}
